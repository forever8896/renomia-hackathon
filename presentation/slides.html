Why This Is The Best Approach

Research-driven architecture with cutting-edge model selection

Research-Backed Decisions

  • MapReduce extraction — "Lost in the Middle" paper (arxiv 2307.03172) shows 30% accuracy drop for info buried in long contexts. We adaptively split at 200K chars.
  • Dual-call ensemble — Multiple papers (arxiv 2511.15714, 2504.18884) show 5-15% accuracy improvement from LLM ensembles. We exploit Gemini's MoE non-determinism.
  • Temperature diversity — "Control the Temperature" (arxiv 2510.01218) shows temp 0 hurts recall. We run 0.0 + 0.15 concurrently.
  • No schema duplication — Google's own docs state duplicating schema in prompt degrades output quality.
  • Few-shot examples — MDPI 2025 reports 91.5% accuracy with few-shot prompt engineering vs lower without.
  • Deterministic ranking — Eliminates 0.15-0.20 score variance from LLM non-determinism across runs.

Competitive Advantages

1 Only Rust solution — fastest cold start, lowest memory
2 Gemini 3.1 Flash Lite — first production use of newest model
3 PDF multimodal fallback — recovers OCR-invisible data
4 Cross-offer attribution — handles real-world data quality
5 Full observability — /analytics endpoint, per-request metrics

Data Quality Finding

Identified and reported a training data error to organizers (lode segment: Pantaenius quotation PDF filed under Allianz). Our cross-offer attribution system detects and handles such cases automatically by scanning for insurer name frequency across offers.
